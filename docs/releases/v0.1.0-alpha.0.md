# v0.1.0-alpha.0 Release Notes

`v0.1.0-alpha.0` is the first technical alpha of Ignition Agent Trainer.

It validates the local framework loop:

```txt
dataset
-> variants
-> rewards
-> experiment runner
-> leaderboard
-> recommendation
-> exporters
-> CLI
-> regression gates
-> IgnitionRAG bridge prototype
```

This tag is a technical alpha, not a product launch. It proves that agent strategies can be measured and compared locally in TypeScript before building hosted IgnitionRAG product surfaces.

## Included Capabilities

- core primitives for datasets, adapters, traces, rewards and experiment results,
- evals and deterministic reward functions,
- experiment runner,
- typed experiment definitions,
- trainer recommendations and objective ranking,
- callable adapter,
- JSON and Markdown exporters,
- local report bundles,
- CLI runner for typed experiment modules,
- regression gates,
- file-based experiment history,
- RAG reward presets,
- strategy preset registry,
- IgnitionRAG adapter contract,
- IgnitionRAG evaluation bridge example,
- policy abstraction,
- trajectory recorder,
- fixed-strategy bandit prototype,
- contextual bandit prototype,
- offline policy evaluation,
- GRPO-style candidate selection,
- PPO interface skeletons.

## What The Alpha Proves

- A developer can define a dataset with expected fragments and citations.
- A developer can compare multiple deterministic agent or RAG variants.
- Rewards can score answer quality, citations, latency and cost.
- Experiments produce a leaderboard and per-case results.
- The trainer can recommend a winning variant and explain tradeoffs.
- Reports can be exported as JSON and Markdown.
- Regression gates can compare current results against committed baselines.
- IgnitionRAG-shaped records can be mapped into `Dataset` and `AgentVariant` objects in a local bridge prototype.

## Explicit Limitations

- No frontend.
- No database.
- No SaaS runtime.
- No real LLM calls by default.
- No model fine-tuning.
- No PPO implementation.
- No GRPO training implementation.
- No production IgnitionRAG integration yet.
- Bandit support remains prototype-level.
- Adapters are minimal and foundational unless a package README documents otherwise.
- IgnitionRAG bridge support is a deterministic local prototype only.

## Validate Locally

Run the full repository checks:

```bash
bun install
bun run lint
bun run typecheck
bun run test
bun run build
```

Run the critical examples:

```bash
bun run --filter './examples/basic-eval' dev
bun run --filter './examples/context-engineering' dev
bun run --filter './examples/callable-adapter' dev
bun run --filter './examples/alpha-dogfood' dev
bun run --filter './examples/ci-regression-gate' dev
bun run --filter './examples/ignitionrag-evaluation-bridge' dev
```

Run the alpha CLI path:

```bash
bun run --filter '@ignitionai/agent-trainer-cli' dev -- eval run ./examples/alpha-dogfood/experiment.ts --bundle reports/alpha-dogfood
```

Generated `reports` and package `dist` folders should not be committed.

## Release Criteria

The alpha is acceptable when:

- `bun run lint` passes,
- `bun run typecheck` passes,
- `bun run test` passes,
- `bun run build` passes,
- all critical examples are runnable,
- there are no open release-blocking PRs,
- `main` is clean,
- tag `v0.1.0-alpha.0` exists and points at the merged alpha bridge commit.

At the time of this release note, the merged #40 alpha bridge commit is included in the pushed `v0.1.0-alpha.0` tag.

## Next Phase

The next phase is not more framework abstraction.

The next phase is dogfooding inside IgnitionRAG:

- IgnitionRAG Evaluation Center prototype,
- Experiment Lab prototype,
- Context Engineering Recommendation view,
- regression check for agent or workflow changes.

Return to this framework only when real IgnitionRAG dogfooding reveals a concrete issue such as:

- missing types,
- awkward APIs,
- reports that are hard to consume,
- CLI friction,
- incomplete adapter contract,
- regression gate ergonomics.

Do not start a new framework feature roadmap until those dogfood findings exist.
